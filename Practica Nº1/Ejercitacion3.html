Ejercitacion 3

3.a)

  • El primero nos muestra el texto dentro del elemento el cual podemos clickear y nos abrira una nueva pagina especificada por el atributo href del elemento.
  • El segundo es muy parecido a primero con la diferencia que el atributo target hace que al clickear el elemento nos va a abrir la pagina pero en una nueva pestaña.
  • El tercero tiene mal escrito el valor del atributo href, especifica el tipo de elemento, lenguaje, caracteres y la relacion del documento con el elemento linkeado.
  • El cuarto solo muestra texto como un link pero al presionarlo no direccion al usuario a ningun lado.
  • El quinto es lo mismo que el cuarto.
  • El sexto no muestra nada en la pagina, es solo el elemento con atributo de nombre e id.

3.b)

  • El primero crea un nuevo parrafo y trata de mostrar una imagen con la descripcion imagen1 que deberia estar en la misma carpeta que el archivo html y llamarse im1.jpg. Tambien se muestra un texto que puede ser clickeado y lleva al usuario a otra pagina, en este caso a google.com.ar
  • El segundo es como el primero pero la imagen que deberia verse puede ser clickeada y redireccionaria al usuario a la pagina de google, ya que este elemento se encuentra añidado dentro del elemento "a". Tambien aparece el texto "click aqui" pero este no puede ser clickeado
  • El tercero es similar al segundo, la unica diferencia es que ahora el elemento añidado de img dentro de "a" tambien contiene el texto por lo que ahora la imagen y el texto son un solo link.
  • El cuarto es similar al tecero, solo que ahora puede clickearse la imagen y el texto por separado, aunque en este caso llevan a la misma pagina.

3.c)

  • El primero muestra una lista sin ordenar con 3 puntos, siendo el primero "xxx", el segundo "yyy" y el tercero "zzz".
  • El segundo muestra una lista ordenada (enumerada) con el mismo orden que el primer caso.
  • El tercero muestra 3 listas ordenadas, donde en la segunda y tercera se especifica el valor de la enumeracion de tal forma que en la pagina parece una sola lista ordenada con tres elementos.
  • El cuarto muestra un bloque resaltado con una lista enumerada de 3 elementos, solo que esta lista no utiliza los elementos de listas sino que esta hecha con un elemento de parrafo y line breaks entre medio de los textos.

3.d)

  • El primero muestra una tabla con 2 columnas las cuales tienen 2 celdas cada una, la primera columna tiene las celdas 1 y 3 y la segunda tiene las celdas 2 y 4. La tabla tiene bordes de 1 pixel y un ancho fijo de 300 pixeles definidos por los atributos border y width.
  • El segundo muestra casi lo mismo, pero el texto de "colummna1" se ve como "colum na1" y ambos textos de columna se encuentran centrados El codigo es parecido al primer caso, con la diferencia de que los elementos de columna contienen el elemento div con atributo de align y valor de centro. Este div es utilizado para centrar el texto de "columna1" y "columna2". Estos textos tambien se encuentran resaltados con negrita con el elemento strong.

3.e)

  • El primero muestra una tabla con un titulo definido por el elemento caption. Contiene 3 columnas con 2 filas de celdas con color de fondo.
  • El segundo es parecido al primero, con la diferencia de que el titulo esta codificado con un colspan de 3 para extenderse a lo largo de 3 celdas. Tambien esta dentro de un div para poder alinearlo en el centro. El resto de las celdas son iguales al caso anterior.

3.f)

  • El primero muestra una tabla de 3 filas, donde la primera fila escribe el titulo juntando 3 columnas, la segunda fila une la primer columna con la tercera fila con rowspan, y las segunda y tercera columna de la segunda y tercera fila se muestran normal.
  • El segundo muestra lo mismo en la primera fila, en la segunda fila une la primera y segunda columna, y el resto se comporta normal.

3.g)

  • El primero muestra una tabla de 3 filas y 3 columnas con un ancho de 200 pixeles y borde de 1 pixel. La primera fila une 3 columnas y muestra "titulo". La segunda fila une 2 filas y columnas en su parte izquierda y una celda normal a su derecha. La tercera fila tiene sus 2 primeras columnas unidas con la segunda fila y su tercera columna con un ancho equivalente al 50% de la tabla, lo que aplica a la columnda inmediata arriba de esta tambien.
  • El segundo muestra una tabla de 3 filas y 2 columnas, con borde de 1 pixel pero sin espacios entre celdas. La primera fila se comporta igual pero con colspan de 2 en vez de 3. La segunda fila simplemente une 2 filas en su primer columna en vez de 2 filas y 2 columnas. La tercer fila se comporta igual, ajustando al 50% el ancho de la columna restante, aunque no seria necesario en este caso

3.h)

  • El primero muestra un formulario con sus respectivos atributos, cuya accion es ejecutar un archivo php. Contiene un fielset donde define dentro un titulo de este "LOGIN" y 2 textos con sus inputs a continuacion, Usuario(tipo texto) y Clave (tipo password), ambos con un valor por default de xxx. Luego del fieldset un input de submit (boton) que envia el formulario.
  • El segundo muestra otro formulario, pero los campos del form no estan dentro de un fieldset, sino que estan ordenados por default y el titulo escrito a mano. Y la clave se ingresa como text en vez de password. También estos campos no cuentan con un valor por default. Otra diferencia es que el form envia datos como variables de URL por el method="get" pero no lo envia a ningun lado ya que action="" esta vacío.
  • El tercero muestra un form que su accion es enviar un mail a una direccion, tiene su fieldset con su legenda, usuario y clave igual que el primero pero sin valor default. El input del final en vez de ser tipo submit es de tipo reset.
